## 프로젝트 목표

고령의 부모님 위치를 가족이 실시간으로 확인하고, 긴급 상황 시 신속하게 대응할 수 있도록 지원하는 반응형 웹 기반 위치 확인 서비스를 구축한다.
디지털 기기에 익숙하지 않은 부모님도 쉽게 사용할 수 있는 간단한 UI를 제공하며, 가족 간 안심 연결 및 긴급 알림 체계를 완성하는 것이 핵심 목표다.

## 핵심 기능

1. 실시간 GPS 위치 공유 (부모님 → 가족)
2. 긴급 알림(SOS) 버튼 — 원터치 누름으로 가족 전원에게 즉시 발송
3. 가족 화면 지도 기반 위치 확인 + 원터치 전화 연결
4. 가족 채팅방 (텍스트, 사진, 위치 카드)
5. 관리자 회원 관리 + SMS/카카오톡 메시지 직접 발송

## 대상 사용자

| 사용자 유형 | 설명 | 주요 니즈 |
|---|---|---|
| **부모님 (피보호자)** | 위치 공유 대상자, 고령자 | 간단한 조작, 긴급 알림 전송 |
| **가족 (보호자)** | 위치 확인 주체 | 실시간 위치 확인, 빠른 연락 |
| **관리자** | 서비스 운영자 | 회원 관리, 고객 커뮤니케이션 |

## 비기능 요건

- 성능: 위치 갱신 딜레이 60초 이내 / SOS 알림 10초 이내 전달
- 보안: 위치 데이터 암호화 전송(HTTPS), 개인정보 처리방침 준수
- 접근성: 반응형 웹(PWA) — 모바일(iOS Safari, Android Chrome), 데스크톱 지원 / WCAG 2.1 AA
- 가용성: 서비스 가동률 99.5% 이상

## 화면 구성 및 기능 상세

### 부모님 화면 (피보호자)

UI 원칙: 큰 글씨체, 높은 명도 대비, 최소 버튼(3개 이하), 한 손 조작 가능

| 기능 | 우선순위 | 설명 |
|---|---|---|
| **긴급 알림(SOS) 버튼** | P0 | 화면 중앙/하단에 크게 배치, 1회 탭으로 가족 전원에게 즉시 발송 |
| 현재 위치 공유 | P0 | 백그라운드 GPS 자동 수집 및 가족에게 공유 |
| 위치 공유 ON/OFF | P1 | 위치 공유 활성화/비활성화 토글 |
| 가족에게 전화 연결 | P1 | 등록된 가족 구성원에게 원터치 전화 연결 |

긴급 알림 플로우:
```
[SOS 버튼 탭] → [확인 팝업 표시 (3초)] → [자동 전송 또는 취소]
→ [가족 전원에게 푸시 알림 + SMS 발송 + 현재 위치 포함]
```

### 가족 화면 (보호자)

UI 원칙: 지도 기반 UI, 전화 연결 및 긴급 알림 수신 항상 노출

| 기능 | 우선순위 | 설명 |
|---|---|---|
| **실시간 위치 확인** | P0 | 지도(카카오맵/네이버맵) 위에 부모님 현재 위치 핀 표시 |
| **긴급 알림 수신** | P0 | SOS 시 즉시 푸시 알림 수신 및 위치 자동 표시 |
| **원터치 전화 연결** | P0 | 부모님에게 바로 전화 연결 버튼 |
| 위치 이력 조회 | P1 | 당일 이동 경로 타임라인 확인 |
| **가족 채팅방** | P1 | 텍스트, 사진, 위치 카드, 읽음 확인 |
| 안심 귀가 알림 | P2 | 자택 반경 진입/이탈 시 자동 알림 (지오펜싱) |
| 배터리 잔량 확인 | P2 | 부모님 기기 배터리 잔량 표시 |

### 관리자 화면

UI 원칙: 데스크톱 최적화, 대시보드 기반 회원 현황 파악

| 기능 | 우선순위 | 설명 |
|---|---|---|
| **회원(가족 그룹) 관리** | P0 | 가입 회원 목록 조회, 상세 정보 확인, 서비스 상태 관리 |
| **SMS 발송** | P0 | 개별 또는 전체 회원에게 문자 메시지 직접 발송 |
| **카카오톡 메시지 발송** | P0 | 카카오 알림톡/친구톡 연동 메시지 발송 |
| 회원 검색 및 필터 | P1 | 이름, 연락처, 가입일 등으로 검색 |
| 긴급 알림 이력 조회 | P1 | SOS 발생 이력 및 처리 현황 확인 |
| 서비스 이용 통계 | P2 | 가입자 수, 활성 사용자, 긴급 알림 현황 대시보드 |
| 공지사항 관리 | P2 | 앱 내 공지사항 등록 및 발송 |

## 기술 요구사항

| 항목 | 내용 |
|---|---|
| 구현 방식 | 반응형 웹 (PWA — 앱 설치 없이 홈화면 추가 가능) |
| 위치 서비스 | GPS (Web Geolocation API), 갱신 주기 30초~1분 |
| 지도 API | 카카오맵 또는 네이버지도 API |
| 푸시 알림 | Web Push (FCM 기반) / PWA Push API |
| SMS | 알리고 또는 NHN클라우드 SMS API |
| 카카오톡 | 카카오 비즈니스 채널 알림톡 API |
| 인증 | 휴대폰 번호 본인인증, 가족 그룹 연결(초대 코드/QR) |

- **Claude Code**: AI 기반 산출물/코드 자동 생성
- **GitHub**: 프로젝트 관리, 이슈 트래킹, CI/CD
- **n8n**: 워크플로우 자동화 (GitHub-Slack 연동, AI Agent)
- **Slack**: 팀 커뮤니케이션, 알림, AI PM Agent
- **NotebookLM**: 프로젝트 위키 + 통합자료실, 산출물/참조자료 통합 조회, AI Q&A
- **Supabase**: PostgreSQL DB + Storage + REST API (권장)
- **Vercel**: 프론트엔드/API 배포 (권장)

## 시작하기

1. `.AP-key.md`에 서비스 URL과 인증 키를 기입합니다
2. `00.통합자료실/`에 프로젝트 참조 자료(RFP, 정책, 인프라 등)를 저장합니다
3. NotebookLM에 프로젝트 노트북을 생성하고, PRD.md + 통합자료실 자료를 소스로 등록합니다
4. Claude Code를 실행합니다: `claude`
5. 첫 명령을 입력합니다: `착수보고서 작성해줘`
