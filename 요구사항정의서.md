## 1. 개요 (Overview)

### 1.1 목적
본 문서는 **부모님 위치 확인 서비스**의 기능적·비기능적 요구사항을 정의한다. 가족 구성원이 부모님의 현재 위치를 실시간으로 확인하고, 부모님이 긴급 상황 시 가족에게 알림을 보낼 수 있으며, 관리자가 회원을 관리·소통할 수 있는 반응형 웹 서비스를 구축하는 것을 목적으로 한다.

### 1.2 서비스 정의
가족에게 부모님의 현재 위치를 실시간으로 제공하는 **반응형 웹(Responsive Web)** 기반 서비스. 부모님·가족·관리자 3개 사용자 그룹별 전용 화면을 제공한다.

### 1.3 대상 사용자
| 구분 | 사용자 | 설명 |
|------|--------|------|
| 부모님 | 위치 제공자 | 위치가 공유되는 대상. 단순 조작 위주 사용 |
| 가족(회원) | 위치 확인자 | 부모님 위치 확인 및 소통 |
| 관리자 | 운영자 | 회원 관리 및 커뮤니케이션 |

### 1.4 핵심 가치
- 부모님의 실시간 위치를 가족이 안심하고 확인
- 긴급 상황 시 부모님이 **단순 조작**으로 가족에게 즉시 알림
- 관리자의 회원 관리 및 다채널 소통

## 2. 시스템 범위 (Scope)

### 2.1 구현 방식
- **반응형 웹** 방식으로 구현 (PC / 태블릿 / 모바일 대응)
- 3개의 사용자 화면: **부모님 화면 / 가족 화면 / 관리자 화면**

### 2.2 화면 구성 요약
```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│  부모님 화면   │   │  가족 화면     │   │  관리자 화면   │
│  - 긴급 알림   │   │  - 위치 확인   │   │  - 회원 정보   │
│  - 단순 조작   │──▶│  - 전화 연결   │   │  - 문자 발송   │
│              │   │  - 가족 채팅   │   │  - 카카오톡    │
└──────────────┘   └──────────────┘   └──────────────┘
```

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 부모님 화면

| ID | 요구사항 | 상세 설명 | 우선순위 |
|------|----------|-----------|----------|
| FR-P-01 | 실시간 위치 전송 | 부모님 단말의 현재 위치(GPS)를 실시간으로 서버에 전송 | 높음 |
| FR-P-02 | 긴급 알림 버튼 | 긴급 상황 시 가족에게 알림을 보내는 버튼 제공 | 높음 |
| FR-P-03 | 단순 조작 알림 | 휴대폰의 단순 조작(버튼 1회 터치 등)만으로 긴급 알림 발송 가능 | 높음 |
| FR-P-04 | 알림 발송 확인 | 알림이 가족에게 정상 전송되었는지 부모님에게 시각적 피드백 제공 | 중간 |

> **UX 고려사항**: 부모님(고령 사용자) 대상이므로 큰 버튼, 큰 글씨, 최소한의 화면 요소로 구성한다.

### 3.2 가족 화면

| ID | 요구사항 | 상세 설명 | 우선순위 |
|------|----------|-----------|----------|
| FR-F-01 | 부모님 위치 확인 | 부모님의 현재 위치를 지도 위에 실시간 표시 | 높음 |
| FR-F-02 | 바로 전화 연결 | 버튼 클릭 시 부모님에게 즉시 전화 연결 | 높음 |
| FR-F-03 | 가족 채팅방 | 가족 구성원 간 채팅 기능 제공 | 중간 |
| FR-F-04 | 긴급 알림 수신 | 부모님의 긴급 알림(FR-P-02)을 즉시 수신·표시 | 높음 |

### 3.3 관리자 화면

| ID | 요구사항 | 상세 설명 | 우선순위 |
|------|----------|-----------|----------|
| FR-A-01 | 회원 정보 확인 | 위치확인 서비스가 연동된 가족(회원) 정보 조회 | 높음 |
| FR-A-02 | 문자(SMS) 발송 | 회원에게 다이렉트로 문자 발송 | 높음 |
| FR-A-03 | 카카오톡 발송 | 회원에게 카카오톡으로 커뮤니케이션 | 중간 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

| ID | 분류 | 요구사항 |
|------|------|----------|
| NFR-01 | 접근성 | 부모님 화면은 고령자 친화적 UI(큰 버튼·글씨, 간결한 구성) 적용 |
| NFR-02 | 반응형 | 모든 화면은 PC/태블릿/모바일 해상도에 대응 |
| NFR-03 | 실시간성 | 위치 정보 및 긴급 알림은 지연 최소화하여 실시간 전달 |
| NFR-04 | 보안 | 위치·개인정보 암호화 및 접근 권한 통제, 개인정보보호법 준수 |
| NFR-05 | 권한 분리 | 부모님/가족/관리자 화면별 접근 권한 분리 |
| NFR-06 | 가용성 | 긴급 알림 기능은 높은 가용성 보장 |

## 5. 외부 인터페이스 (External Interfaces)

| 구분 | 연동 대상 | 용도 |
|------|-----------|------|
| 지도 API | 지도 서비스(예: 카카오맵/네이버맵/Google Maps) | 위치 표시 |
| 위치(GPS) | 단말 위치 정보 | 실시간 위치 수집 |
| 전화 연결 | 단말 통화 기능(tel:) | 부모님 바로 전화 |
| SMS | 문자 발송 서비스 | 관리자 → 회원 문자 |
| 카카오톡 | 카카오 알림톡/메시지 API | 관리자 → 회원 카톡 |

## 6. 가정 및 제약사항 (Assumptions & Constraints)

| ID | 내용 |
|------|------|
| AC-01 | 부모님 단말은 위치(GPS) 권한이 허용되어 있어야 함 |
| AC-02 | 부모님과 가족은 사전에 서비스에 연동(회원 매핑)되어 있어야 함 |
| AC-03 | SMS·카카오톡 발송은 외부 발송 대행 서비스 계약을 전제로 함 |
| AC-04 | 반응형 웹 기반이며 별도 네이티브 앱은 본 범위에서 제외 |
