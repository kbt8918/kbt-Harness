# 화면설계서 (Screen Design Specification)

**프로젝트명**: 부모님 위치 확인 서비스
**작성일**: 2026-06-09
**버전**: v1.0
**작성자**: kbt8918 (기획자)
**근거 자료**: 기능명세서.md(v1.0), 정보구조도.md(v1.0)

> 본 문서는 정보구조도의 사이트맵·사용자 흐름과 기능명세서의 단위 기능(F-001~F-014)을 화면별 레이아웃·UI 컴포넌트·인터랙션으로 전개한다.
> 본 문서의 결과는 인프라아키텍처(#10)·디자인스타일가이드(#13)와 Week 4 프론트엔드 코드 생성(`Next.js + Tailwind CSS`)의 입력 자료가 된다.
> 역할별 화면은 권한 분리(F-012)에 따라 분기되며, 모든 화면은 반응형 레이아웃(F-014)으로 PC/태블릿/모바일에 대응한다.

## 화면 상세

### SCR-001. 로그인

| 항목 | 내용 |
|------|------|
| URL | `/login` |
| 플랫폼 | 모바일/PC 공통 (반응형) |
| 접근 권한 | 비로그인 사용자 |
| 관련 기능 | F-011(로그인·인증), F-012(권한 분리·역할 분기) |
| 관련 API | `POST /api/auth/login`, `POST /api/auth/logout` |

```
+-------------------------------+  375px
|         Status Bar            |

|     부모님 위치 확인 서비스      |
|                               |
+-------------------------------+
|                               |
|  아이디(이메일)                 |
|  +-------------------------+  |
|  |                         |  |
|  +-------------------------+  |
|                               |
|  비밀번호                      |
|  +-------------------------+  |
|  |                    [눈]  |  |
|  +-------------------------+  |
|                               |
|  [ ] 자동 로그인               |
|                               |
|  +-------------------------+  |
|  |        로그인            |  |
|  +-------------------------+  |
|                               |
|  (로그인 후 역할별 자동 분기:    |
|   부모님 / 가족 / 관리자)       |
|                               |
+-------------------------------+
```

**UI 컴포넌트**

| 컴포넌트 | 타입 | 설명 | 인터랙션 |
|----------|------|------|----------|
| 아이디 입력 | TextInput | 이메일 형식 | 포커스 시 라벨 상단 이동, 형식 미준수 시 인라인 에러 |
| 비밀번호 입력 | TextInput (password) | 마스킹, bcryptjs 해시 검증 | 눈 아이콘 탭 시 표시/숨김 토글 |
| 자동 로그인 | Checkbox | 토큰 유지 여부 | 체크 시 세션 토큰 보존 |
| 로그인 버튼 | Button (primary) | 인증 요청 CTA | 탭 시 `POST /api/auth/login`, 성공 시 역할 분기, 실패 시 에러 토스트 |

**상태별 화면**

| 상태 | 조건 | 표시 내용 |
|------|------|-----------|
| 기본 | 진입 | 입력 폼 표시 |
| 로딩 | 인증 요청 중 | 로그인 버튼 스피너, 입력 비활성 |
| 에러 | 자격 불일치 | "아이디 또는 비밀번호를 확인해주세요" 토스트 |
| 차단 | 미인증 접근/토큰 만료 | 401 안내 + 로그인 유도 (F-012) |

| 위치 권한 거부 | Geolocation 거부 | 안내·재요청 + 서비스 사용 제한 고지 (F-001) |
| GPS 미수신 | 좌표 수집 실패 | 마지막 위치 유지 + "위치 확인 불가" 표시 |
| 전송 실패 | 네트워크 끊김 | 재전송 큐 동작 + "전송 실패, 다시 시도" (F-002) |
| 세션 만료 | 인증 만료 | 단순 재로그인 안내 |

## 공통 컴포넌트 가이드

### 모달 / 다이얼로그

```
+-------------------------------+
|  [제목]                  [X]  |
|  -------------------------    |
|  내용 영역 (설명/폼 요소)       |
|  -------------------------    |
|           [취소]   [확인]     |
+-------------------------------+
```

### 토스트 알림

```
모바일: 하단 중앙, 3초 후 자동 닫힘 / 클릭 시 즉시 닫힘
PC:     우측 상단, 3초 후 자동 닫힘
용도:   로그인 실패, 발송 결과, 전송 실패 등 (성공=중립, 에러=경고색)
```

### 긴급 알림 강조 (F-005)

```
+----------------------------------------------------------------------+
| [!] 긴급 알림 - OOO님이 긴급 알림을 보냈습니다 (OO:OO)    [위치보기]  |
+----------------------------------------------------------------------+
  - 일반 알림과 구분되는 강조 색상 + 사운드
  - 가족 화면(SCR-003) 상단 고정 노출, [위치보기] 클릭 시 지도 포커스
```

### 빈 상태 (Empty State)

```
+-------------------------------+
|        [일러스트/아이콘]        |
|     데이터가 없습니다           |
|     안내 메시지                |
|        [액션 버튼]             |
+-------------------------------+
```

### 권한 차단 (F-012)

```
+-------------------------------+
|        [잠금 아이콘]           |
|   접근 권한이 없습니다 (403)    |
|   요청이 차단되고 기록되었습니다 |
|        [홈으로]               |
+-------------------------------+
```

## 반응형 설계 원칙 (F-014)

| 플랫폼 | 기준 해상도 | 설계 원칙 |
|--------|------------|-----------|
| 모바일 | 375px | 부모님 화면(SCR-002)은 모바일 우선·핵심 버튼 유지, 단일 컬럼 |
| 태블릿 | 768px | 가족 화면 2단 → 1단 축소, 지도 우선 |
| PC | 1920px | 가족 화면 지도+정보 2단, 관리자 화면 사이드바+테이블 |

- Tailwind 반응형 브레이크포인트(`sm`/`md`/`lg`/`xl`)로 화면별 레이아웃 분기
- 부모님 화면은 저사양 단말에서도 긴급 버튼 우선 렌더, 비정상 해상도 시 최소 레이아웃 보장
