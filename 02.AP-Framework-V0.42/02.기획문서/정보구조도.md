# 정보구조도 (Information Architecture)

**프로젝트명**: 부모님 위치 확인 서비스
**작성일**: 2026-06-09
**버전**: v1.0
**작성자**: kbt8918 (기획자)
**근거 자료**: 기능명세서.md(v1.0), 요구사항정의서.md(v1.0)

> 본 문서는 기능명세서의 화면(부모님/가족/관리자)과 단위 기능(F-001~F-014)을 사이트맵·사용자 흐름·화면-기능 매핑으로 구조화한다.
> 본 문서의 결과는 화면설계서(#9)의 입력 자료가 된다.
> 역할별 진입은 권한 분리(F-012)에 따라 분기된다: 부모님 / 가족 / 관리자.

## 1. 전체 사이트맵

```mermaid
flowchart TD
    L[로그인 / 인증 F-011] --> R{역할 분기 F-012}

    R -->|부모님| P[부모님 화면]
    R -->|가족| F[가족 화면]
    R -->|관리자| A[관리자 화면]

    P --> P1[고령자 UI 대시보드 F-003]
    P1 --> P2[긴급 버튼 1-터치 F-002]
    P1 --> P3[위치 자동 전송 F-001]
    P1 --> P4[발송 상태 피드백 F-002]

    F --> F1[부모님 위치 지도 F-004]
    F --> F2[실시간 위치·알림 수신 F-005]
    F --> F3[바로 전화 F-006]
    F --> F4[가족 채팅 F-007]

    A --> A1[회원 조회 F-008]
    A --> A2[SMS 발송 F-009]
    A --> A3[카카오톡 알림톡 발송 F-010]
    A --> A4[부모-가족 매핑 관리 F-013]
```

> 모든 화면은 반응형 레이아웃(F-014)으로 PC/태블릿/모바일에 대응하며, 부모님 화면은 모바일 우선·핵심 버튼 유지.

## 2. 사용자 흐름 (User Flow)

### 2.1 긴급 알림 흐름 (부모님 → 가족)

```mermaid
flowchart LR
    Start([부모님 화면 진입]) --> Btn[긴급 버튼 1회 터치 F-002]
    Btn --> Dbnc{중복 연타?}
    Dbnc -->|Yes| One[디바운스: 1건 처리]
    Dbnc -->|No| Send[긴급 알림 API 호출]
    One --> Send
    Send --> Map{매핑 가족 존재?}
    Map -->|No| Admin[관리자 통보]
    Map -->|Yes| Cast[실시간 브로드캐스트 + 위치 첨부]
    Cast --> Fb[부모님 화면 전송됨 피드백 F-002]
    Cast --> Recv[가족 화면 긴급 알림 강조 표시 F-005]
    Fb --> End([완료])
    Recv --> End
    Send -->|네트워크 실패| Retry[재시도 → 실패 시 다시 시도 표시]
    Retry --> End
```

### 2.2 실시간 위치 확인 흐름 (부모님 → 가족)

```mermaid
flowchart LR
    GPS([부모님 단말 GPS]) --> Perm{위치 권한?}
    Perm -->|거부| Guide[안내·재요청, 사용 제한 고지 F-001]
    Perm -->|허용| Collect[주기마다 좌표 수집 F-001]
    Collect --> Post[POST /api/location]
    Post --> Save[(서버 저장)]
    Save --> Stream[실시간 채널 브로드캐스트 /api/stream]
    Stream --> Marker[가족 지도 마커 갱신 F-004/F-005]
    Marker --> View([가족이 위치 확인])
    Post -->|네트워크 끊김| Queue[재전송 큐]
    Queue --> Post
```

### 2.3 바로 전화 흐름 (가족)

```mermaid
flowchart LR
    Click([바로 전화 버튼 클릭 F-006]) --> Chk{매핑·권한 확인}
    Chk -->|불허| Block[403 차단]
    Chk -->|허용| Phone{전화번호 등록?}
    Phone -->|미등록| Disable[버튼 비활성 + 안내]
    Phone -->|등록| Tel[tel: 링크 실행]
    Tel --> Call([단말 통화 앱 발신])
```

### 2.4 관리자 발송 흐름 (회원 관리)

```mermaid
flowchart LR
    Login([관리자 로그인 F-011]) --> List[회원 조회 F-008]
    List --> Select[수신 대상 선택]
    Select --> Ch{발송 채널}
    Ch -->|SMS| Sms[POST /api/admin/sms F-009]
    Ch -->|카카오톡| Kakao[POST /api/admin/kakao F-010]
    Sms --> Res[발송 결과 수집·기록]
    Kakao --> Res
    Res --> Report([성공/실패 건수 표시])
    Sms -->|사업자 API 오류| Retry[재시도·실패 기록]
    Kakao -->|미승인 템플릿| Stop[차단]
    Retry --> Report
```

### 2.5 인증·권한 분리 흐름 (공통)

```mermaid
flowchart TD
    Req([보호 리소스 요청]) --> Auth{인증 토큰 유효?}
    Auth -->|No| E401[401 로그인 유도]
    Auth -->|Yes| Role{역할·매핑 검증 F-012}
    Role -->|위반| E403[403 차단 + 접근 로깅]
    Role -->|허용| Resource[리소스 응답]
```

## 3. 화면-기능 매핑

| 화면명 | URL(예시) | 주요 기능 | 관련 기능 ID |
|--------|-----------|-----------|-------------|
| 로그인 | `/login` | 로그인·인증, 역할 분기 | F-011, F-012 |
| 부모님 화면 | `/parent` | 고령자 UI, 긴급 알림, 위치 전송, 발송 피드백 | F-001, F-002, F-003 |
| 가족 화면 | `/family` | 위치 지도, 실시간 수신, 바로 전화, 가족 채팅 | F-004, F-005, F-006, F-007 |
| 가족 채팅방 | `/family/chat/:roomId` | 가족 채팅 송수신 | F-007 |
| 관리자 화면 | `/admin` | 회원 조회, SMS·카카오 발송, 매핑 관리 | F-008, F-009, F-010, F-013 |
| (공통 레이아웃) | 전체 | 반응형 레이아웃, 권한 분리 | F-014, F-012 |
